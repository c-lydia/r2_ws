# Contributing — Docs & PR checklist

Guidelines for contributing documentation and code to this repository.

## Documentation updates

- Add new or updated Markdown files under `docs/`.
- Update `docs/index.md` with any new top-level pages.
- Keep content concise and include commands in code blocks or indented blocks.
- Add diagrams to `docs/img/` and reference them from the Markdown.

## PR checklist

- [ ] Branch named like `docs/<short-description>` or `feature/<ticket>`
- [ ] Include a short description of changes in PR body
- [ ] Update `docs/index.md` if you added/removed pages
- [ ] Update `CHANGELOG.md` with user-facing changes (if applicable)
- [ ] Run linters and unit tests (where applicable)
- [ ] Provide screenshots for UI/diagram changes
- [ ] Request at least one reviewer

## Style notes

- Use imperative, present-tense verbs for headings and instructions.
- Prefer step-by-step examples for commands.
- When adding code examples, include minimal, copyable commands.
